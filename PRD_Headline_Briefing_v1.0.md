# Project Headline Briefing - PRD 초안 (Product Requirements Document)

## 1. 제품 개요 (Product Overview)
**Headline Briefing**은 국내외 주요 언론사의 헤드라인 뉴스를 주제별로 정리하여, 사용자가 바쁜 일상 속에서도 세상의 주요 이슈들을 한눈에 파악할 수 있도록 돕는 '뉴스 브리핑 앱'입니다.

## 2. 목표 및 타겟 유저
- **주요 목표:** 정보의 홍수 속에서 사용자가 관심 있는 주제의 최고 핵심(Headline)만 효율적으로 소비할 수 있는 환경 제공.
- **타겟 유저:** 
  - 출퇴근 시간, 자투리 시간에 뉴스를 빠르게 파악하고 싶은 직장인 및 학생.
  - 국내 이슈뿐만 아니라 해외 주요 언론(NYT, BBC 등)의 소식을 글로벌하게 모니터링하고 싶은 사용자.

## 3. 핵심 기능 (Core Features) - MVP 

### 3.1. 주제별 뉴스 피드 
- **주제 분류:** 종합, 정치, 경제, 사회, IT/과학, 해외/세계 등 직관적인 탭 구성.
- **리스트업:** 각 탭 내에서 국내외 주요 언론사의 최신 헤드라인 리스트 제공.

### 3.2. 일일 요약 브리핑 (Daily Briefing)
- **맞춤형 푸시(Push):** 아침/저녁 등 사용자가 설정한 시간대에 주요 기사를 모아 푸시 알림 발송.
- **AI 요약:** 단순히 제목만 보여주는 것이 아니라, 기사 내용의 핵심을 3줄 내외로 요약하여 바쁜 유저의 시간 절약.

### 3.3. 해외 뉴스 번역 보조
- 해외 주요 언론사의 뉴스 헤드라인과 요약을 **한국어로 번역**하여 제공 (원문/번역 토글 지원).

### 3.4. 언론사/키워드 필터링 및 북마크
- **필터링:** 사용자가 신뢰하거나 선호하는 언론사만 선택하여 피드를 구성.
- **북마크:** 중요하거나 나중에 깊게 읽고 싶은 뉴스를 저장.
- **키워드 알림:** 관심 키워드(예: "금리", "AI", "테슬라")를 등록하면 관련 속보 발생 시 알림.

## 4. 기술 스택 제안 (Tech Stack Requirements)

가장 빠르고 효율적으로 MVP를 구현하기 위한 모던 기술 스택 제안입니다.

### 4.1. 클라이언트 (Mobile App)
- **React Native (Expo):** iOS와 Android를 단일 코드베이스로 동시 지원 가능. 기존 웹 개발 경험을 살려 빠르게 제작 가능.
- **상태 관리:** Zustand 또는 React Query (서버 데이터 캐싱 및 동기화에 유리).

### 4.2. 데이터 저장 및 제공 (Backend-less Architecture)
- **정적 JSON 호스팅:** 별도의 DB 서버 구축 없이(Supabase 배제), 뉴스 데이터를 JSON 파일(`latest_news.json`)로 생성하여 정적 배포(GitHub Pages 등)로 호스팅.
- **클라이언트 단독 저장소:** 사용자의 북마크, 설정 등은 기기 내부 로컬 DB(`AsyncStorage` 또는 `SQLite`)에만 저장하여 개인정보를 보호하고 서버 비용을 없앰.

### 4.3. 데이터 수집(Data Pipeline) 및 AI 요약
- **뉴스 소스:** 언론사 RSS Feed 파싱, 혹은 무료 News API 활용.
- **수집 파이프라인:** **GitHub Actions (Cron Job)** 활용. 매시간 스크립트를 실행하여 최신 기사를 수집.
- **AI 요약:** 파이프라인 스크립트에서만 한 번 OpenAI API 등을 호출해 요약/번역을 수행. (이후 정적 파일로 저장하여 AI API 추가 비용 방지).

---

## 5. 시스템 아키텍처 및 데이터 플로우

1. **데이터 수집기 (GitHub Actions)** 
   - 지정된 주기(예: 매 1시간)에 최신 기사 수집 파이썬/Node 스크립트가 가상 환경에서 실행됨.
2. **AI 프로세싱 및 정적 파일 생성**
   - 수집된 기사를 바탕으로 **헤드라인 추출 -> 핵심 3줄 요약 -> 번역** 과정을 거친 뒤 그 결과물을 모아 `latest_news.json`으로 저장.
3. **정적 배포 (Static Hosting)**
   - 생성된 JSON 파일을 GitHub 저장소(gh-pages 브랜치 등)에 푸시 및 배포.
4. **클라이언트 렌더링 (Client-Alone)**
   - 모바일 앱은 호스팅된 JSON URL만 호출하여 데이터를 고속으로 가져옴. 유저별 데이터(북마크)는 로컬에 캐싱 및 저장.

---

## 6. 개발 단계별 마일스톤

- **Phase 1 (데이터 파이프라인 구축):** 
  - 뉴스 데이터 수집 및 OpenAI 연동 스크립트(Python 기반) 개발.
  - GitHub Actions를 통한 `news.json` 정적 호스팅 자동화 구축.
- **Phase 2 (핵심 앱 기능 개발):**
  - React Native (Expo) 프로젝트 초기화.
  - 배포된 JSON 데이터를 Fetch하여 캐싱하는 레이어 구현 및 메인/카테고리 피드 화면 개발.
  - 뉴스 요약본 제공 상세 모달 구현.
- **Phase 3 (고도화 및 부가 기능):**
  - 로컬 스토리지 연동(북마크), 언론사 필터링 기능.
  - 앱 스토어 심사 및 배포.
